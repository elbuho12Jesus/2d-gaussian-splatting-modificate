Todo verificado contra el código en su estado actual (no estimado): scene/gaussian_model.py, gaussian_renderer/__init__.py, train.py, arguments/__init__.py y el rasterizer CUDA submodules/diff-surfel-rasterization/cuda_rasterizer/{forward.cu, backward.cu}. Las líneas citadas son las reales a fecha 2026-06-23.

0 · La idea en una ecuación

En 2DGS clásico, la opacidad de un splat sobre un píxel decae con una gaussiana de la distancia de Mahalanobis al cuadrado ρ. El kernel Beta sustituye esa gaussiana por una potencia con exponente aprendible β por splat:

2DGS clásico (gaussiana)
α = opa · exp(−½ρ)

Cola infinita (nunca llega a 0). Forma fija. forward.cu:397,412 (ahora comentado como OLD CODE).

Beta Splatting
α = opa · (1 − ρ)β

Soporte compacto (vale 0 exacto en ρ≥1). β deforma la curva: β grande = pico estrecho casi-gaussiano; β pequeño = meseta plana que cubre todo el disco. forward.cu:405,416.

Comparación visual del kernel (1−ρ)β según β frente a la gaussiana:

Curvas esquemáticas. Clave: con β bajo (naranja, el valor al que converge el entrenamiento) el kernel es casi plano — cubre el disco entero con opacidad casi uniforme y cae a plomo en el borde. Con β alto (azul) concentra la masa en el centro como la gaussiana pero con corte limpio.

_beta es el parámetro crudo (raw, en log-espacio), entrenable, con shape (N,1) — un β por gaussiana. Se inicializa a 0.

Como la activación es 4·exp(b), el valor crudo 0 arranca en β = 4 (un pico bastante concentrado, casi gaussiano). El entrenamiento luego lo baja hasta una media cruda ≈ −3.45 → β ≈ 0.13 (mesetas planas). El modelo prefiere kernels casi planos: dato clave para la sección de mejoras.

2Activación: de crudo a β efectivo PY

scene/gaussian_model.py:144-147

@property
def get_beta(self):
    b = self._beta.clamp(min=-6.0, max=2.0)
    return (4.0 * torch.exp(b)).contiguous()
β = 4 · exp( clamp(b, −6, 2) )

El kernel CUDA nunca ve el crudo b, ve siempre el β activado. La exponencial garantiza β > 0 (necesario: el exponente de (1−ρ)β debe ser positivo) y el clamp acota el rango antes de exponenciar.

4Forward CUDA: el corazón del kernel CUDA

submodules/diff-surfel-rasterization/cuda_rasterizer/forward.cu:378-417

float2 s = {p.x / p.z, p.y / p.z};            // intersección rayo–splat, Eq.(10) 2DGS
float rho3d = (s.x * s.x + s.y * s.y);        // ρ en el plano local del surfel
float2 d = {xy.x - pixf.x, xy.y - pixf.y};
float rho2d = FilterInvSquare * (d.x*d.x + d.y*d.y);   // filtro paso-bajo (2·‖d‖²)
float rho = min(rho3d, rho2d);                // ρ = min(3D, 2D)

float r2 = rho;
float beta_j = collected_beta[j];
if(r2 >= 1.0f) continue;                      // soporte compacto: fuera del disco
float one_minus_r2 = max(1.0f - r2, 1e-6f);   // guard anti log(0)/pow
float kernel = powf(one_minus_r2, beta_j);    // ★ K = (1 − ρ)^β
...
float alpha = min(0.99f, opa * kernel);       // ★ α = min(0.99, opa·(1−ρ)^β)
if (alpha < 1.0f / 255.0f) continue;
ρ = min(ρ3D, ρ2D)  ∈ [0,1)   |   K(ρ,β) = (1 − ρ)β   |   α = min(0.99, opa · K)
  • ρ es la distancia de Mahalanobis al cuadrado: ρ=0 en el centro del splat, crece hacia el borde. El min con ρ2d es el filtro paso-bajo que evita aliasing en splats diminutos.
  • Soporte compacto (if(r2>=1.0f) continue): a diferencia de la gaussiana, el kernel Beta vale 0 exacto fuera del disco ρ<1. No hay cola.
  • El OLD CODE comentado justo encima (l.396-414) es la gaussiana original exp(−½ρ) — útil para A/B futuro.

5Backward CUDA: los tres gradientes CUDA

submodules/diff-surfel-rasterization/cuda_rasterizer/backward.cu:319-421 · :496 · :437-460

De α = opa·(1−ρ)β salen tres derivadas. El código usa formas estables (reescritas en términos de α para no recalcular potencias):

5.1 · Gradiente respecto a β

backward.cu:406-415

if (beta_j >= 0.1f) {                          // ← umbral anti-NaN (solo afecta a β)
    float d_alpha_d_beta = alpha * logf(one_minus);      // ∂α/∂β = α·log(1−ρ)
    float grad_beta = dL_dalpha * d_alpha_d_beta;
    grad_beta = fminf(fmaxf(grad_beta, -1e-3f), 1e-3f);  // ← clamp ±0.001
    atomicAdd(&dL_dbeta[global_id], grad_beta);
}
∂α/∂β = α · log(1 − ρ)

Como log(1−ρ) < 0 siempre, subir β baja α (más pico = menos cobertura en los bordes). Dos diques de contención que limitan cuánto se entrena β: (1) el umbral β≥0.1 congela el gradiente por debajo (evita log(1−ρ)→−∞ con NaN), y (2) el clamp ±1e-3 recorta el paso. Ambos son candidatos de la sección de mejoras.

5.2 · Gradiente respecto a ρ (geometría)

backward.cu:417-421 · :437-460

// dα/dρ = −β·α/(1−ρ)   (forma estable, equivale a −opa·β·(1−ρ)^{β−1})
float d_alpha_d_rho = (-beta_j) * alpha / one_minus;
...
float dL_d_rho = dL_dalpha * d_alpha_d_rho;
const float2 dL_ds = { dL_d_rho*2.0f*s.x + dL_dz*Tw.x,
                       dL_d_rho*2.0f*s.y + dL_dz*Tw.y };   // ∂ρ/∂s = 2s
∂α/∂ρ = −β · α / (1 − ρ)   |   ∂ρ/∂s = 2s

Este gradiente empuja la posición/escala/rotación del surfel (vía T). El factor 2s es la derivada de ρ=s·s — en la gaussiana iba implícito en −G·s; con Beta hay que ponerlo explícito (FIX #1 de run9).

5.3 · Gradiente respecto a la opacidad

backward.cu:327 · :496

const float G = kernel;                        // G = (1−ρ)^β
...
atomicAdd(&(dL_dopacity[global_id]), G * dL_dalpha);   // ∂α/∂opa = (1−ρ)^β

Por qué β≥0.1 NO congela el resto: el FIX #2 (run9) movió el umbral para que solo salte el atomicAdd de β. Antes un continue con β<0.1 saltaba todo el cuerpo del bucle y congelaba posición/escala/opacidad de ese splat. Hoy esos tres gradientes (5.2, 5.3) se calculan siempre. β se entrena con Adam, lr=0.001, sin regularización explícita (solo el L1 indirecto de la foto-loss). Hay dos clamps distintos: el de entrenamiento [−4, 2] (train.py) y el de lectura [−6, 2] (get_beta). El efectivo es el más estrecho: β ∈ [0.073, 29.6].

Coincidencia importante: el clamp inferior de entrenamiento es b=−4 → β=0.073, que está por debajo del umbral de gradiente β=0.1 (b≈−3.69). Es decir, hay una franja β∈[0.073, 0.1] donde β puede estar pero no recibe gradiente → queda congelado ahí. Y la media observada (β≈0.13) está pegada a esa franja.

El −log(N) del split (l.531) no es arbitrario: con la activación exponencial, 4·exp(b−logN) = (4·exp b)/N, así N hijos suman el β del padre y la opacidad acumulada del disco se conserva tras partir.

Existe además un filtro de densificación por β opcional y desactivado por defecto (gaussian_model.py:473-488, flags beta_densify_threshold=0.0 / beta_densify_mode="split_wide" en arguments/__init__.py:132-133): permitiría splitear solo splats por encima/debajo de un β umbral. Nunca se ha usado en un run.

Palancas del kernel aún sin explorar

Los 5 sweeps fallidos del historial (cap / gate / opacity_reg / scale_reg / gate-medio) nunca tocaron β directamente. Estas son palancas del propio kernel, ordenadas por relación coste/novedad. Recordatorio del diagnóstico: el velo del MCMC = floaters translúcidos sobre-brillantes de fondo; β bajo = mesetas planas que cubren el disco entero con opacidad casi uniforme → sospechoso directo del velo.

A · Soltar los diques de β riesgo bajo · barato

Hoy β apenas se entrena: clamp de gradiente ±1e-3 (backward.cu:412) + beta_lr=0.001. Quizá β no ha elegido la meseta, sino que no ha podido moverse de su arrastre inicial. Prueba: subir beta_lr (0.001→0.005) y/o aflojar el clamp (±1e-3→±5e-3). Mide si la media cruda se aleja de −3.45 y si el velo cede. Requiere recompilar el rasterizer solo si tocas el clamp CUDA.

B · Regularización de β hacia picos riesgo medio · barato

No existe término que regule β. Si el velo viene de mesetas (β bajo), añadir en train.py (junto a opacity_reg/scale_reg, l.156-157) un beta_reg · relu(β_target − β).mean() que empuje β hacia arriba (kernels más concentrados, menos cobertura translúcida de borde). Es la versión "kernel" del anti-velo, frente a los reg de opacidad/escala ya agotados. Riesgo: empujar demasiado endurece bordes y reabre huecos (modo de error del clásico).

C · Revisar el umbral β≥0.1 y el clamp inferior riesgo medio

Hay una franja muerta β∈[0.073, 0.1] donde β existe pero no recibe gradiente (§6), y la media está pegada a ella. Posible trampa unidireccional: β cae a la franja y se congela plano. Prueba: subir el clamp inferior de entrenamiento a b=−3.5 (β≥0.12, fuera de la franja) o bajar el umbral CUDA a 0.05 con un guard log más robusto. Recompila rasterizer.

D · Init de β menos agresivo riesgo bajo · barato

Arranca en β=4 (pico) y el entrenamiento lo desploma a 0.13. Si la dinámica de descenso es el problema, inicializar el crudo en, p.ej., −1.4 (β≈1, lineal) podría evitar el sobre-disparo a meseta. Cambio de una línea (gaussian_model.py:192).

E · Activar el filtro de densificación por β riesgo medio

Ya está cableado pero a 0 (§7, gaussian_model.py:473-488). beta_densify_mode="split_wide" + threshold≈0.2 splitearía solo las mesetas anchas — densificar donde el kernel es plano para reemplazar un velo grande por varios splats más definidos. Solo aplica al modo clásico.

F · Acoplar β a la escala del surfel riesgo alto · estructural

El techo del rasterizer (run21): los gigantes del fondo están clavados en s(max)=extent y siguen translúcidos. Un β que dependa del tamaño (forzar β alto en surfels grandes → su meseta se vuelve pico, menos translucidez de borde) atacaría justo a esos gigantes que scale_reg no pudo mover. Requiere lógica nueva en forward/backward; es la palanca más cara pero la única que apunta al techo identificado.

Recomendación de orden: empezar por A y D (baratos, una línea, sin recompilar en el caso de beta_lr) para ver si β se mueve cuando le quitas los frenos. Si moverlo cambia el velo, escalar a B (regularización dirigida). F solo si A/B confirman que β es la palanca del fondo. Todo medido honesto-vs-honesto contra run16 (0.339 / 0.597 / 20.21).
