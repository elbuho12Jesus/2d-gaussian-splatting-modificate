Reciclar splats muertos:
relocate_gs línea por línea

Cómo el MCMC rellena los huecos que dejan los splats que se apagan — copiando splats vivos de alto error sobre los slots muertos, sin crear memoria nueva y sin cambiar la transmitancia total de la escena.

líneas 678–735 llama a _error_signal · _reset_optimizer_state sin jitter propio N constante

El MCMC no crea splats en relocate_gs: los recicla. Un gate previo marca como muertos los splats de opacidad ínfima (dead_mask). Esta función copia encima de esos slots un splat vivo elegido por sorteo —sesgado hacia opacidad alta y zonas mal reconstruidas— para no desperdiciar capacidad. El número total de splats N no cambia: es una relocalización, no un crecimiento (eso lo hace add_new_gs).

La clave sutil está en la opacidad: al duplicar un splat, su opacidad se reparte entre las copias con una fórmula que conserva exactamente cuánta luz bloquea el conjunto. Lo vemos abajo con números.

AGuardas de entrada

Líneas 686–693 — no hacer nada si no hay a quién reciclar o de dónde copiar.

n_dead = int(dead_mask.sum().item())   # ¿cuántos muertos?
if n_dead == 0: return                 # nadie que reubicar → salir
alive_mask = ~dead_mask
dead_idx  = dead_mask.nonzero()[0]      # índices de los MUERTOS (destino)
alive_idx = alive_mask.nonzero()[0]     # índices de los VIVOS (candidatos a fuente)
if alive_idx.shape[0] == 0: return      # no hay de dónde copiar → salir

dead_mask es un booleano de tamaño N. .nonzero() lo convierte en dos listas de índices: dead_idx (los slots que se van a sobrescribir) y alive_idx (los que pueden servir de plantilla). Las dos guardas evitan trabajo inútil o divisiones por cero.

BPesos de muestreo: opacidad × error

Líneas 695–705 — elegir qué splats vivos se clonan, por sorteo ponderado.

alive_op = get_opacity[alive_idx]                     # peso base = opacidad
if error_weight > 0.0:
    err = self._error_signal()                        # error de reconstrucción por splat
    alive_op = alive_op * (1 + error_weight * err[alive_idx])
weights = alive_op / alive_op.sum()                   # normaliza a probabilidad
src_idx = alive_idx[multinomial(weights, n_dead, replacement=True)]

El peso de cada splat vivo empieza siendo su opacidad (los sólidos y confiables pesan más). Con error_weight>0 —tus runs usan mcmc_error_weight = 3.5— se multiplica por (1 + 3.5·err), donde err viene de _error_signal(): el gradiente de viewspace acumulado, normalizado a media ≈ 1. Traducción: los splats que están en zonas mal reconstruidas atraen más copias.

multinomial(..., replacement=True) sortea n_dead fuentes con reemplazo → un mismo splat popular puede salir varias veces. Ese detalle es el que obliga a la fórmula de la sección C.

No es "el más opaco" Es un sorteo aleatorio ponderado, no un argmax. Un splat de opacidad baja pero en zona de alto error todavía puede ganar; uno opaco en zona ya resuelta puede no salir nunca. Los splats sin señal de error reciben err = 0 y caen a muestreo por opacidad pura.

CConservación de transmitancia

Líneas 714–717 — repartir la opacidad para que el conjunto bloquee la misma luz.

ratio = bincount(src_idx)[src_idx]                    # nº de veces que salió cada src
src_alpha = get_opacity[src_idx].clamp(2e-3, 1-1e-3)
new_alpha = 1 - (1 - src_alpha)**(1/(ratio+1))        # ← la fórmula clave
new_opacity_raw = inverse_opacity_activation(new_alpha)

Si un splat de opacidad α se elige ratio veces, acabará existiendo en ratio+1 instancias: las copias en los slots muertos más el original. Apilar k splats de opacidad a deja pasar una fracción de luz (1−a)ᵏ. Para que k = ratio+1 instancias bloqueen lo mismo que una de opacidad α, se exige:

(1 − new_alpha)ratio+1 = (1 − α)   ⟹   new_alpha = 1 − (1 − α)1/(ratio+1)

Y a ambos lados se les asigna esa misma new_alpha: la copia (línea 726) y el original (línea 727). Por eso el splat fuente baja su propia opacidad al ser clonado — no se duplica masa translúcida, se reparte.

Docstring desactualizado La línea 682 dice new_alpha = src/2. Es la versión vieja: solo es correcta si ratio == 1 (un único clon) y sobre-infla la opacidad de los src populares. El código real usa la fórmula ratio+1 de arriba (comentario 708–713). El docstring quedó sin actualizar.

DLa copia

Líneas 719–727 — volcar todos los atributos del src sobre el slot muerto.

_xyz[dead_idx]      = _xyz[src_idx]     # MISMA posición (sin jitter)
_features_dc / _rest / _sb_params ...   # color y spherical-betas
_scaling[dead_idx]  = _scaling[src_idx] # forma / tamaño
_rotation[dead_idx] = _rotation[src_idx]
_beta[dead_idx]     = _beta[src_idx]    # perfil del kernel β
_opacity[dead_idx]  = new_opacity_raw   # copia → opacidad repartida
_opacity[src_idx]   = new_opacity_raw   # original → también baja

La copia es exacta salvo la opacidad: posición, color, escala, rotación y β se clonan tal cual. Nótese la línea 719: _xyz se copia idéntico — el clon nace encima del original.

¿Y el jitter? relocate_gs no aplica jitter. La separación entre el clon y su fuente la produce el paso de ruido MCMC posterior en train.py (bloque [NOISE]), que sacude a los splats de opacidad baja. Como ambos nacen con new_alpha reducida, el ruido los alcanza y los dispersa.

EReseteo de contadores y del optimizador

Líneas 729–735 — que el splat reubicado empiece "de cero".

low_opacity_counter[dead_idx] = 0   # reinicia el gate de muerte
xyz_gradient_accum[dead_idx]  = 0   # olvida el error del splat anterior
denom[dead_idx] = 0 ; max_radii2D[dead_idx] = 0
reset_idx = unique(cat(dead_idx, src_idx))
_reset_optimizer_state(reset_idx)   # momentos de Adam (exp_avg / exp_avg_sq) → 0

El slot reubicado hereda la geometría del src pero no su historial: los contadores se ponen a cero para que no lo maten inmediatamente ni arrastre gradientes ajenos. Además se resetea el estado de Adam (medias móviles) tanto de los destinos como de las fuentes (reset_idx = unión de ambos), porque su opacidad acaba de saltar y el momento viejo ya no aplica.

La aritmética de la opacidad

El splat 1 salió ratio = 2 veces → ratio+1 = 3 instancias. Aplicamos la fórmula de la sección C con α = 0.80:

ratio = bincount([1,1])[1] = 2 new_alpha = 1 − (1 − 0.80)1/3 = 1 − 0.200.3333 = 1 − 0.5848 = 0.4152 # comprobación de transmitancia (luz que deja pasar): una instancia original: 1 − 0.80 = 0.2000 tres instancias nuevas: (1 − 0.4152)³ = 0.5848³ = 0.2000 ✓

La escena bloquea exactamente la misma luz que antes: no hemos añadido ni quitado opacidad neta, solo la hemos redistribuido en 3 puntos que el ruido posterior dispersará para cubrir el hueco de alto error. Nótese que el splat 3 (α 0.05) nunca fue elegido: peso ínfimo. Y reset_idx = unique([2,4,1,1]) = [1,2,4] → se resetea Adam en esos tres.

En una frase relocate_gs sortea splats vivos (por opacidad × error), los copia idénticos sobre los muertos, y reparte la opacidad de cada fuente entre sus instancias para no alterar la transmitancia total — dejando que el ruido MCMC posterior los separe. Sin jitter, sin crear splats, N constante.
